============================================================
ENVIRONMENT COMMANDS
============================================================

Default environment comes from config/runSettings.json:
"defaultEnvironment": "TEST"

Report auto-open flags in config/runSettings.json:
"openPlaywrightReport": true
"openExcelReport": true

Run one environment:
npm run register -- --env TEST --site ALL --product ALL
npm run register -- --env PROD --site ALL --product ALL

Environment shortcuts:
npm run register:test -- --site NJ --product ALL
npm run register:prod -- --site NVRCP --product RPM

Run the same sites/products on two environments:
npm run register -- --env TEST,PROD --site NJ --product CL,BB
npm run register -- --env TEST,PROD --site NJ,NVRCP --product ALL --parallel 3

Run different sites on different environments:
npm run register -- --target TEST:NJ,PROD:NVRCP --product ALL --parallel 3
npm run register -- --target TEST:DPBH,PROD:SAPTA --product ALL

List without executing:
npm run register:list -- --env PROD --site ALL --product ALL
npm run register:list -- --env TEST,PROD --site NJ --product ALL
npm run register:list -- --target TEST:NJ,PROD:NVRCP --product ALL

Notes:
- PROD has no CRANES URL, so PROD + CRANES is not generated.
- PROD NJ uses version 11.3.24, so only CL and BB are generated.
- TEST NJ uses version 11.4.38.05, so CL, BB, HMB, and ESF are generated.
- Update environment URLs only in config/urls.js when a URL changes from .05 to .06.
- Credentials are saved under testData\<ENVIRONMENT>\ to prevent TEST/PROD overwrite within the same run.
