==========================

Note:
  The full Register -> Apply flow is kept in serial order.
  --parallel controls Playwright workers, but the default full flow still runs Script 1 and Script 2 together in one Playwright run so the HTML report includes both scripts.
  The HTML report opens automatically after the selected run finishes.


Standalone Script Commands
==========================

Run only Script 1 registration:
  npm run test --spec=register --env=TEST --headed
  npm run test --spec=01_RegisterUser.spec.js --env=PREPROD --project=chromium

Run only Script 2 login/apply:
  npm run test --spec=apply --env=TEST --headed
  npm run test --spec=02_LoginApply.spec.js --env=PROD --project=chromium

For standalone Script 2 credentials, edit:
  tests\LNI\Conv_E2E\config\script2LoginCredentials.js

If loginName/password are blank in that file, Script 2 uses the latest saved user:
  tests\LNI\Conv_E2E\testData\userData.json

Login / Entity Name Format
==========================

Login names use this format in TEST, PRE PROD, and PROD:
  Conv_CC_12
  Conv_CC_123
  Conv_CC_1234

The first registration attempt uses 2 digits.
If the login name is taken, retry attempts use 3 digits, then 4 digits, then 5 digits, and so on.
The maximum number of attempts is controlled here:
  runSettings.json -> registrationLoginRetryLimit

Entity names use this format in TEST, PRE PROD, and PROD:
  CONV_DD-MM-YYYY_HH:MM:SS AM/PM

Example:
  CONV_25-06-2026_09:54:12 AM
